# TempoGuitar 开发说明

## 1. 项目定位

TempoGuitar 是一个面向个人练琴场景的桌面级节拍器应用，核心目标是提供一个稳定、直观、低打扰、适合长期打开使用的练习工具。

它不是通用型 DAW，也不是复杂的编曲工具，而是一个围绕节奏训练展开的轻量桌面应用，重点服务以下使用场景：

- 日常基础节拍训练
- 不同 BPM 下的速度爬升练习
- 切分、重音与拍号感知训练
- 针对吉他练习过程中的循环练习与阶段训练

## 2. 产品目标

### 核心目标

- 提供准确、稳定、响应迅速的节拍器体验
- 支持适合练琴的常用参数快速调整
- 兼顾“即开即用”和“逐步进阶”的使用方式
- 以桌面应用形态提供更稳定的常驻使用体验

### 非目标

- 首个版本不做复杂录音、编曲、谱面编辑
- 首个版本不做云同步依赖
- 首个版本不追求过多社交或社区功能

## 3. 用户画像

主要用户是会长期练琴的乐手，后续也可以覆盖以下人群：

- 吉他练习者
- 贝斯练习者
- 鼓手或节奏训练用户
- 需要稳定节拍参考的器乐学习者

## 4. 建议技术栈

### 桌面应用框架

- Tauri 2.x
- Rust
- WebView 前端壳
- 后续可接入 `window-vibrancy` 做 Windows 11 Mica 增强

### 前端

- Vue 3
- Vue Router
- TypeScript
- Vite
- CSS Variables + 自定义样式系统
- `<script setup>` 组合式 API

### 为什么优先选择 Vue 3

- 更贴合当前熟悉的技术栈，能更快进入功能开发
- 组合式 API 很适合拆分节拍器、音频调度、预设管理等逻辑
- SFC 对界面与交互组织更直观，适合快速迭代桌面工具型产品
- 与 Vite、Tauri 结合成熟，初始化和开发体验较顺畅

### 状态与数据

- Vue 组合式状态管理优先
- 后续如复杂度提升，可引入 Pinia
- 本地持久化优先使用 Tauri Store 或本地文件配置

### 音频与节拍调度

- Web Audio API 负责节拍声音播放
- `AudioContext` + 高精度调度逻辑实现点击音播放
- Rust 端优先负责应用能力、配置与系统集成，不在第一阶段承载核心音频逻辑

### 构建与质量

- ESLint
- TypeScript strict mode
- Prettier
- Vitest
- Playwright（后续可选）

## 5.1 后续视觉增强方案

为了让应用更接近原生桌面工具质感，后续可以评估接入 `window-vibrancy`，在 Windows 11 上启用 Mica（云母）窗口效果。

### 方案定位

- 属于后续增强项，不阻塞 MVP 开发
- 仅作为 Windows 11 的平台增强能力
- 非 Windows 11 环境自动降级为普通背景

### 方案原则

- 不将透明材质作为界面可用性的前提
- 即使关闭 Mica，界面也必须保持完整、清晰、可读
- 优先保证窗口稳定性、拖动体验与性能表现

### 接入注意事项

- 需要评估 Tauri 窗口透明配置与实际效果
- 前端样式需要兼容透明背景与普通背景两种模式
- 需要重点验证启动、拖拽、缩放、聚焦等窗口行为
- 如出现平台兼容性问题，应优先保留普通窗口方案

## 6. MVP 功能范围

第一阶段建议只做最重要、最能直接提升练琴体验的部分。

### 必须具备

- 开始 / 暂停节拍
- BPM 调节
- 拍号设置
- 重音设置
- 音量控制
- 节拍音色区分（强拍 / 弱拍）
- 可视化拍点反馈
- 常用参数本地保存

### 强烈建议加入

- Tap Tempo
- BPM 快捷增减
- 键盘快捷键
- 最近使用配置
- 支持常见细分节奏
- 支持最高到 16 分音符的基础细分练习
- 支持多种点击音色和基础鼓声音色

### 第二阶段功能

- 练习预设保存
- 分阶段提速练习
- 自动 BPM 递增
- 小节循环计数
- 练习计时器
- 自定义重音模式
- 更复杂的节奏型训练

### 后续增强功能

- 训练模式统计
- 多种点击音包
- MIDI 设备联动
- 全局悬浮控制
- 系统托盘常驻
- 导入 / 导出练习预设
- 基础鼓机能力

## 7. 交互设计原则

这个产品是高频工具，不应像营销页面，而要像练习台上的一件器材。

设计原则建议如下：

- 核心操作必须在 1 到 2 次点击内完成
- 大号 BPM 显示，远距离也能看清
- 开始 / 暂停按钮必须醒目且低误触
- 参数面板清晰分组，避免信息拥挤
- 优先支持键盘操作与快速微调
- 在练习过程中避免无关弹窗和打断

## 8. 信息架构建议

## 10. 模块拆分建议

### metronome

负责 BPM、拍号、播放状态、当前拍位置、调度时钟与声音触发。

建议以 Vue composable 的形式抽离核心逻辑，例如 `useMetronome()`。

### audio engine

负责 Web Audio 初始化、点击音缓冲、前瞻调度、时间漂移控制。

建议拆成独立的音频服务层，例如 `src/lib/audio/`，避免与界面组件耦合。

### presets

负责保存用户常用节拍配置，例如：

- 80 BPM 4/4 基础练习
- 120 BPM 八分音符切分练习
- 逐级提速练习模板

### practice session

负责练习计时、阶段切换、自动加速与后续统计功能。

适合逐步抽成独立 store 或 composable，避免早期过度设计。

## 11. 开发阶段建议

### Phase 0：项目初始化

- 初始化 Tauri + Vue 3 + TypeScript + Vite
- 配置 lint、format、基础目录结构
- 搭建基础 UI 框架

### Phase 1：节拍器核心 MVP

- BPM 设置
- 播放控制
- 拍号 / 重音
- 基础声音播放
- 本地保存配置

### Phase 2：练习增强

- Tap Tempo
- 练习预设
- 自动提速
- 计时器

### Phase 3：桌面能力增强

- 系统托盘
- 快捷键
- 窗口置顶
- 更完整的设置页

### Phase 4：节奏扩展

- 复杂节奏训练
- 更丰富的鼓声音色
- 基础鼓机功能

### Phase 5：桌面视觉增强

- 评估 `window-vibrancy`
- Windows 11 下启用 Mica
- 完善普通背景回退方案
- 验证窗口拖动、缩放与启动体验

## 12. 开发优先级建议

优先把“准、稳、顺手”做出来，再去做更花哨的功能。

建议优先级如下：

1. 高精度节拍调度
2. 清晰直观的核心交互
3. 本地保存与快速恢复
4. 16 分音符与常用细分训练支持
5. 多种点击音和鼓声音色
6. 练习增强能力
7. 桌面系统集成能力
