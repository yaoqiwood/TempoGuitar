# TempoGuitar 开发说明

## 1. 项目定位

TempoGuitar 是一个面向个人练琴场景的桌面级节拍器应用，核心目标是提供一个稳定、直观、低打扰、适合长期打开使用的练习工具。

它不是通用型 DAW，也不是复杂的编曲工具，而是一个围绕节奏训练展开的轻量桌面应用，重点服务以下使用场景：

- 日常基础节拍训练
- 不同 BPM 下的速度爬升练习
- 切分、重音与拍号感知训练
- 针对吉他练习过程中的循环练习与阶段训练

## 2. 产品目标

### 核心目标

- 提供准确、稳定、响应迅速的节拍器体验
- 支持适合练琴的常用参数快速调整
- 兼顾“即开即用”和“逐步进阶”的使用方式
- 以桌面应用形态提供更稳定的常驻使用体验

### 非目标

- 首个版本不做复杂录音、编曲、谱面编辑
- 首个版本不做云同步依赖
- 首个版本不追求过多社交或社区功能

## 3. 用户画像

主要用户是你自己这类会长期练琴的乐手，后续也可以覆盖以下人群：

- 吉他练习者
- 贝斯练习者
- 鼓手或节奏训练用户
- 需要稳定节拍参考的器乐学习者

## 4. 建议技术栈

### 桌面应用框架

- Tauri 2.x
- Rust
- WebView 前端壳

### 前端

- Vue 3
- Vue Router
- TypeScript
- Vite
- CSS Variables + 自定义样式系统
- `<script setup>` 组合式 API

### 为什么优先选择 Vue 3

- 更贴合你当前熟悉的技术栈，能更快进入功能开发
- 组合式 API 很适合拆分节拍器、音频调度、预设管理等逻辑
- SFC 对界面与交互组织更直观，适合快速迭代桌面工具型产品
- 与 Vite、Tauri 结合成熟，初始化和开发体验较顺畅

### 状态与数据

- Vue 组合式状态管理优先
- 后续如复杂度提升，可引入 Pinia
- 本地持久化优先使用 Tauri Store 或本地文件配置

### 音频与节拍调度

- Web Audio API 负责节拍声音播放
- `AudioContext` + 高精度调度逻辑实现点击音播放
- Rust 端优先负责应用能力、配置与系统集成，不在第一阶段承载核心音频逻辑

### 构建与质量

- ESLint
- TypeScript strict mode
- Prettier
- Vitest
- Playwright（后续可选）

## 5. MVP 功能范围

第一阶段建议只做最重要、最能直接提升练琴体验的部分。

### 必须具备

- 拍号设置

- Tap Tempo

- 键盘快捷键
- 最近使用配置
- 深色/浅色之外的专注练习主题

### 第二阶段功能

- 练习预设保存
- 分阶段提速练习
- 自动 BPM 递增
- 小节循环计数
- 练习计时器

### 后续增强功能

- 训练模式统计
- 多种点击音包
- MIDI 设备联动
- 全局悬浮控制
- 系统托盘常驻
- 导入 / 导出练习预设

## 6. 交互设计原则

这个产品是高频工具，不应像营销页面，而要像练习台上的一件器材。

设计原则建议如下：

- 核心操作必须在 1 到 2 次点击内完成
- 大号 BPM 显示，远距离也能看清
- 开始 / 暂停按钮必须醒目且低误触
- 参数面板清晰分组，避免信息拥挤
- 优先支持键盘操作与快速微调
- 在练习过程中避免无关弹窗和打断

### 扩展面板

## 9. 模块拆分建议

### metronome

负责 BPM、拍号、播放状态、当前拍位置、调度时钟与声音触发。

建议以 Vue composable 的形式抽离核心逻辑，例如 `useMetronome()`。

### audio engine

负责 Web Audio 初始化、点击音缓冲、前瞻调度、时间漂移控制。

建议拆成独立的音频服务层，例如 `src/lib/audio/`，避免与界面组件耦合。

### presets

负责保存用户常用节拍配置，例如：

- 80 BPM 4/4 基础练习
- 120 BPM 八分音符切分练习
- 逐级提速练习模板

### practice session

负责练习计时、阶段切换、自动加速与后续统计功能。

适合逐步抽成独立 store 或 composable，避免早期过度设计。

## 10. 开发阶段建议

### Phase 0：项目初始化

- 初始化 Tauri + Vue 3 + TypeScript + Vite
- 配置 lint、format、基础目录结构
- 搭建基础 UI 框架

### Phase 1：节拍器核心 MVP

- BPM 设置
- 播放控制
- 拍号 / 重音
- 基础声音播放
- 本地保存配置

### Phase 2：练习增强

- Tap Tempo
- 练习预设
- 自动提速
- 计时器

### Phase 3：桌面能力增强

- 系统托盘
- 快捷键
- 窗口置顶
- 更完整的设置页

## 11. 开发优先级建议

优先把“准、稳、顺手”做出来，再去做更花哨的功能。

建议优先级如下：

1. 高精度节拍调度
2. 清晰直观的核心交互
3. 本地保存与快速恢复
4. 练习增强能力
5. 桌面系统集成能力

## 12. 当前环境结论

当前机器已具备：

- Node.js
- npm

当前机器暂未检测到：

- Rust / Cargo

因此下一步要正式初始化 Tauri 工程，需要先安装 Rust 工具链，然后再执行 Tauri 脚手架创建命令。

## 13. 下一步建议

建议按下面顺序推进：

1. 安装 Rust 工具链
2. 初始化 Tauri Vue 3 TypeScript 项目
3. 实现节拍器最小可运行界面
4. 实现高精度节拍播放逻辑
