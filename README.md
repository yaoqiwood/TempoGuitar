# TempoGuitar

TempoGuitar 是一个面向日常练琴场景的桌面级节拍器应用，计划基于 `Tauri + Vue 3 + TypeScript + Vite` 技术栈构建。

这个项目的目标不是做一个复杂的音乐制作软件，而是做一个真正适合乐手长期打开、随手就能开始练习的节拍器工具。它会优先关注准确、稳定、顺手，以及练琴场景下最常用的能力。

## 项目目标

TempoGuitar 主要服务于日常练琴，尤其适合吉他、贝斯和其他需要节奏训练的器乐练习者。

核心目标：

- 节拍稳定，声音触发准确
- 界面直观，能快速开始和调整参数
- 适合长时间放在桌面上使用
- 支持基础练习和逐步进阶练习

非目标：

- 首个版本不做 DAW、编曲、录音类复杂能力
- 首个版本不做社交和社区功能
- 首个版本不依赖云端服务

## 使用场景

这个应用主要面向以下练习场景：

- 日常基础节拍练习
- 从慢速到高速的爬格子或分段提速练习
- 不同拍号下的节奏稳定性训练
- 强拍、弱拍、切分和重音感知训练
- 预设常用练习方案并快速恢复

## 核心功能设想

下面这部分是当前建议功能清单，后面可以一起删减成更聚焦的 MVP。

### 一、基础节拍器能力

- 开始 / 暂停播放
- BPM 调节
- 拍号设置
- 小节内当前拍显示
- 重音设置
- 强拍 / 弱拍不同音色
- 音量控制
- 节拍闪烁或视觉反馈
- 支持常见细分节奏显示与播放
- 支持最高到 16 分音符级别的基础细分练习

### 二、适合练琴的增强能力

- Tap Tempo
- BPM 快速加减
- 键盘快捷键控制
- 最近一次参数自动恢复
- 练习预设保存
- 常用参数一键切换
- 支持更复杂的节奏组合与重音变化练习
- 支持不同细分类型下的练习切换

### 三、进阶练习功能

- 自动提速
- 分阶段练习
- 小节循环计数
- 练习计时器
- 自定义重音模式
- 不同训练模式切换
- 复杂节奏型训练
- 后续可扩展为更完整的鼓机模式

### 四、桌面应用增强能力

- 系统托盘常驻
- 窗口置顶
- 快捷启动和快速控制
- 本地配置持久化
- 后续可考虑全局快捷键

### 五、声音与节奏扩展能力

- 支持多种节拍音色
- 支持更接近鼓组感觉的点击声
- 支持鼓声音色，包括适合练琴的基础鼓点声音
- 后续可扩展为不同风格的声音包
- 为未来接入鼓机功能预留结构

## 建议的 MVP 范围

如果我们希望先尽快做出一个“已经能每天拿来练琴”的版本，我建议第一版只保留这些：

- 开始 / 暂停
- BPM 设置
- 拍号设置
- 重音设置
- 强拍 / 弱拍音色区分
- 音量调节
- 当前拍视觉反馈
- 本地保存最近使用参数

如果你希望第一版再多一点实用性，也可以把下面两项一起纳入：

- Tap Tempo
- 练习预设保存
- 基础细分节奏支持（包括到 16 分音符）

## 暂时不建议首版就做的功能

这些功能不是没价值，而是容易把项目拉复杂，建议放到第二阶段以后：

- 录音
- 谱面编辑
- 云同步
- 账号系统
- 社交分享
- MIDI 深度集成
- 多端同步
- 完整鼓机编辑能力
- 复杂鼓组编排界面

## 产品设计原则

这个应用应该像练习桌上的一个可靠工具，而不是一个花哨的展示页面。

设计上建议坚持这些原则：

- 主要操作在 1 到 2 次点击内完成
- BPM 数字必须足够大，远一点也能看清
- 开始 / 暂停按钮必须明显
- 练习时不要出现无关打扰
- 参数布局清晰，不要堆太多信息
- 能键盘操作的尽量支持键盘操作

## 技术栈

当前建议技术方案如下：

- Tauri 2.x
- Vue 3
- TypeScript
- Vite
- Web Audio API
- Rust

选择 Vue 3 的原因：

- 更符合你熟悉的技术栈，可以更快进入开发状态
- Vue 3 组合式 API 很适合拆分节拍器逻辑
- 单文件组件对工具型桌面应用的界面组织很直观
- 与 Vite、Tauri 的组合比较成熟

## 计划中的模块划分

建议后续按这些模块拆分：

- `metronome`
  负责 BPM、拍号、播放状态、拍点推进
- `audio engine`
  负责 Web Audio 初始化、点击音播放与高精度调度
- `presets`
  负责常用练习配置保存与切换
- `practice session`
  负责计时、阶段训练与自动提速
